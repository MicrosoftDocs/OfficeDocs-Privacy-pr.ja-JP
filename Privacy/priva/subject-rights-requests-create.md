# <a name="create-a-subject-rights-request"></a>サブジェクト権限要求を作成する

サブジェクト権限管理管理者は、Priva のメインの [件名の権利要求] ページから新しい要求を開きます。 ウィザードは、データ主体に関する個人データを検索し、要求を満たすプロセスを開始するプロセスを案内します。

また、追加の資料をアップロードして、Priva が提供された正確なデータ値に基づいてデータ主体を識別することもできます。 詳細については、「Subject [Rights Requests のデータ一致」を参照してください](subject-rights-requests-data-match.md)。

## <a name="request-types"></a>要求の種類

Priva Subject Rights Requests は、次の 3 種類の要求をサポートしています。

1. **アクセス**: 組織内で組織が保持するデータ主体の個人情報の概要をMicrosoft 365。

2. **エクスポート**: データ主体の個人情報を含むコンテンツ アイテムの概要とエクスポートファイルを提供します。 これらは、検索設定によって収集されたデータのレビュー中に確認され、含まれるとマークされたアイテムです。

3. **フォロー用のタグ付きリスト**: データ レビュー中にタグ付けされたファイルの概要が生成され、Priva の外部で追加のアクションが必要な場合があります。 たとえば、データ主体の要求に応じて、データ主体の個人情報の削除を容易にする必要がある場合があります。 組み込みタグを表示し、Priva 設定で組織のカスタム タグ [を設定できます](priva-settings.md)。

## <a name="use-the-subject-rights-request-creation-wizard"></a>件名の要求作成ウィザードを使用する

1. [権限 [] Microsoft 365 コンプライアンス センター](https://compliance.microsoft.com/)左側のナビゲーションから **[Priva Subject Rights Requests**] を選択します。

2. 右上隅で、[要求の作成 **] を選択します**。

3. [データ主体 **情報] ページ** で、データ主体の名と名、電子メール アドレス、居住国を追加し、会社との関係を指定します。 [**次へ**] を選択します。

4. [場所 **] ページ** で、データMicrosoft 365の個人データを探す場所を選択します。 場所の設定に関 [する詳細を取得します](#choose-locations)。 場所の選択が完了したら、[次へ] を **選択します。**

5. [検索設定 **] ページ** で、検索設定を変更して検索を絞り込み、データを取得する前に最初に見積もりを取得するかどうかを選択できます。 このページでオプションを選択する必要がない場合は、前の手順で指定した場所に基づいて既定の検索が行います。 ここでのオプションの詳細については、「検索設定の [定義」を参照してください](#define-search-settings)。 続行する準備ができたら、**[次へ]** を選択します。

6. [要求 **の種類] ページ** で、[](#request-types)データ主体がデータに対して実行する操作に基づいて要求の種類を選択 **します。[** アクセス]、[エクスポート]、または [タグ付きリスト] を選択して **、フォローアップを行います**。  要求が特定のデータ プライバシー規制に関連する場合は、指定されたリストから要求を選択して、コンテキストを追加できます。 期限 (必須) を設定すると、要求に近づくか、期限が過ぎた場合に簡単に並べ替え、それらを簡単に解決できます。 完了したら、[次へ] を **選択します**。

7. [要求 **名] で**、この要求の名前を確認または編集し、省略可能な説明を参照用に追加します。 [**次へ**] を選択します。

8. [確認 **と終了] ページ** で、選択内容を確認します。 任意のセクションを編集できます。 完了したら、[要求の作成] **を選択します**。

要求には独自の詳細ページが表示され、メインの [件名の権利要求] ページに一覧表示されます。 要求を作成すると、Priva は、指定した場所で指定したデータ主体の情報に一致する情報の検索を開始します。 検索範囲によっては、要求の詳細ページにデータ見積もりを表示するには数分かかる場合があります。

## <a name="choose-locations"></a>場所を選択する

サブジェクト権限要求ごとに、検索を設定して、すべての場所または特定の場所にあるデータを検索Exchange Sharepoint に設定できます。 トグル スイッチを On 位置に移動して場所 **を選択** します。 すべてのアカウントとサイトを検索するか、各場所内の特定のアカウントまたはサイトを指定することができます。 各場所で説明されている内容の詳細については、以下をご覧ください。

- **Exchange**: このオプションは、メールボックス内のデータExchange、個々のチャットまたはグループ チャットTeamsします。 組織内のすべてのアカウントを検索Exchange、または [アカウントの選択] を選択して、[メールボックス] フライアウト ウィンドウから **個々のユーザー Exchange** 選択できます。

- **SharePoint**: このオプションは、サイト、サイト、SharePointチャネルOneDrive for BusinessデータをTeamsします。 組織内のすべてのサイトを検索SharePoint、または [サイトの選択] を選択して、[サイト] フライアウト ウィンドウから個々の **ユーザー SharePoint** 選択できます。

適切な検索用語の識別に関するヘルプについては、次のトピックを参照してください。

- **SharePointサイトと URL**: [SharePoint](/sharepoint/manage-sites-in-new-admin-center) 管理センターでサイトを管理すると、サイトを並べ替え、フィルター処理する方法、およびサイトを検索する方法に関するガイダンスがSharePointされます。 これを使用して、[サイト] フライアウト ウィンドウの検索フィールドに入力 **SharePoint URL** を検索します。

- **Teamsチャットと** チャネル: [Get-Team](/powershell/module/teams/get-team) は、特定のプロパティまたは情報を提供することで、Microsoft Teamsチームを検索する方法を示します。

- **OneDriveサイトと URL**: [OneDrive URL](/onedrive/list-onedrive-urls#about-onedrive-urls) については、ユーザーのユーザーの URL の適切な形式とプロパティに関する情報をOneDriveします。 これを使用して、検索内のOneDriveを特定するのに役立ちます。

適切なデータ主体を特定するために、選択内容を慎重に確認することをお勧めします。 たとえば、名前でメールボックスを検索し、類似する名前を持つ複数の個人を見つけた場合は、要求にメールボックスを追加する前に正しいメールボックスを確認してください。

## <a name="define-search-settings"></a>検索設定の定義

サブジェクト権限要求を作成すると、作成ウィザードの [場所] ページの選択内容に基づいて既定の検索が実行されます。 より対象を絞った検索を実行する場合、またはコンテンツ アイテムを取得する前にデータの推定値を取得する場合は、要求作成ウィザードの [検索設定] ページでそれらの選択を行います。

### <a name="advanced-search-options"></a>高度な検索オプション

- **検索を絞り** 込む: このオプションを使用すると、組織のデータの中でデータ主体を特定するための追加のプロパティを指定できます。 このオプションを選択すると、検索パラメーターの追加を求めるメッセージが表示されます。詳細については、「検索の絞り込み [」で説明します](#refine-your-search)。
- **データ主体によって作成された** コンテンツを含める: このオプションは、データ主体によって作成されたコンテンツを検索します。 たとえば、データ主体によって作成またはアップロードSharePointファイルが含まれます。 このオプションを選択すると、返されるデータ量が大幅に増加する可能性があります。
- **すべてのバージョンの** アイテムを含める: 検索場所として [SharePoint] を選択した場合、既定の検索クエリは現在のバージョンのアイテムのみをSharePointします。 このボックスをオンにすると、現在および以前のすべてのバージョンのアイテムSharePointが返され、確認するデータ量が大幅に大きくなります。

### <a name="data-estimate"></a>データの見積もり

[ **最初の推定値を取得** する] オプションは、データが自動的に取得される前に見つけるデータ量の推定値を示します。 要求の詳細ページに見積もりが表示されている場合は、検索結果を表示し、検出されたアイテムのサンプリングをプレビューできます。 アイテムが予想される結果を表す場合は、[データの取得] を選択して、コンテンツ アイテムの実際の取得を続行する必要があります。

## <a name="refine-your-search"></a>検索を絞り込む

[検索設定] **ページ** で [検索の絞り込み] を選択した場合、[次へ] を選択すると、検索結果をさらに対象とする個人の属性と条件の詳細を入力するように求めるメッセージが表示されます。 いずれかのカテゴリまたは両方のカテゴリに追加できます。 このセクションで選択が完了すると、要求のデータ取得は検索設定に基づいて行われます。

#### <a name="add-personal-attributes"></a>個人属性の追加

データ主体の名前、ニックネーム、電子メール アドレス、およびアドレスのテキスト フィールドに値を入力します。 生年月日や電話番号などの他の識別子を追加し、テキスト フィールドはセミコロンで区切られた複数のエントリをサポートします。 完了したら、[次へ] を **選択して** [条件] ページに **進** む。

#### <a name="conditions"></a>条件

[条件の **追加** ] ボタンを選択して、アイテム名、送信者と受信者の名前、個人データの種類、アイテムが組織外で外部で共有されたかどうかなど、検索をさらに対象とする条件の範囲を選択します。 テキスト フィールドは、セミコロンで区切られた複数のエントリをサポートします。 完了したら、[次へ **] を選択** して検索設定を保存し、要求の種類の設定に進む。
