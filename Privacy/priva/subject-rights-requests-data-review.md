# <a name="review-data-for-a-subject-rights-request"></a>サブジェクト権限要求のデータを確認する

Microsoft Priva でサブジェクト権限[要求 (詳細](subject-rights-requests-create.md)を参照) を作成すると、Subject Rights Requests ソリューションは入力を使用して、組織の Microsoft 365 環境でデータ主体に関する一致を検索します。 このデータがコンパイルされると、結果を確認し、何を含めるかについて選択し、必要に応じて情報を編集できます。 これらの手順は、Priva インターフェイスを介して複数のユーザーが共同作業できます。

## <a name="step-1-review-request-details-and-monitor-progress"></a>手順 1: 要求の詳細を確認し、進行状況を監視する

検索の最初の結果を表示するには、ユーザーの [権限] の [Priva] 領域に移動しMicrosoft 365 コンプライアンス センター **を開きます**。[](https://compliance.microsoft.com/) 開いているすべてのサブジェクト権限要求の一覧は、このメイン ページで確認できます。

一覧で要求を選択すると、要求の詳細が表示されます。 ここでは、要求のプロパティ、検索結果、要求の状態について詳しくは、こちらを参照してください。 このページは、見つかったファイルの管理、レポートとエクスポートの作成、要求の完了に関する作業と共同作業を行うハブになります。

要求の詳細ページのタイルには、次のものが含まれます。

- **詳細**: 要求の期限と要求日、説明、関連するプライバシー規制など、要求に関する重要な詳細。
- **進行状況**: 完了した手順と、まだ完了していないタスクを示すタイムライン。
- 現在の進捗ステージに関する統計情報。 このタイルには、データ見積の概要、検索で見つかったアイテムの数、Microsoft 365 の場所、エクスポートの状態など、情報が表示される場合があります。
- **確認する優先度の高い** アイテム: このタイルには、Priva が検出した重要なアイテムに関する情報が表示されます。 たとえば、既に Microsoft の機密ラベルが付いた機密情報や、やり直しが必要な複数の個人に関するデータを含むアイテムが含まれる場合があります。 これにより、管理者はレビューの開始場所を知るのに役立ちます。 優先度の項目は、[優先度の種類] 列でフィルター処理して収集されるデータの下に表示されます。

### <a name="understand-progress-stages"></a>進行状況のステージを理解する

サブジェクト権限要求は、複数のステージを通過します。 一部の状態は自動的に進行し、サブジェクト権限が管理者や投稿者にファイルの確認などの重要な手順を完了すると、他のステージが進みます。

要求は時間のかかる場合や複数の投稿者が作業する必要がある場合がありますから、Priva は、次に実行する手順に関する状態とガイダンスに関する継続的な更新を提供します。 これらの更新プログラムは、サブジェクト **権限要求の** 詳細ページの [概要] タブで表示できます。

#### <a name="data-estimate"></a>データの見積もり
要求を作成すると、Priva は、ユーザー環境内のデータ主体に一致する可能性のあるMicrosoft 365開始します。 条件に一致すると思うすべてのアイテムが特定されると、要求の [概要] ページの [データ見積もり概要] カードに見積もりが **表示** されます。 検索範囲内のデータの量は、見積もりの完了にかかる時間に影響します。

要求はデータ取得の次の段階に自動的に移動し、すべてのコンテンツ アイテムが収集され、関係者がデータレビューで共同作業できます。 場合によっては、取得に移行する前にデータ見積もりを一時停止し、続行する前に実行する次の手順を通知します。

また、サブジェクト権限要求を最初に作成するときに、データ推定段階で自動的に一時停止することもできます。 作成プロセス中に、[検索設定] ステップ **の間に** [最初に見積もりを取得する **] オプションを選択** します。 検索設定手順の [詳細を確認します](subject-rights-requests-create.md#define-search-settings)。

#### <a name="pause-in-data-estimate-for-large-search-results"></a>大規模な検索結果のデータ見積もりを一時停止する

Priva は、レビューするアイテムの数が多い (10K を超えるアイテム) データ見積もりが予想される場合に通知されます。 推定は一時停止し、結果をプレビューし、検索クエリを編集して[](subject-rights-requests-create.md#refine-your-search)、より特定の場所や条件をターゲットにするか、特定のアイテムを取得し続けるかを決定します。  検索に一致するアイテム数とデータ量を画面に表示します。 画面の上部にあるメッセージ バーには、次のオプションの 1 つまたは両方があります。

- [ **検索クエリの編集** ] ボタンを使用すると、要求の検索設定に直接移動して、より厳密なパラメーターを設定し、新しい見積もりを生成します。
- 検索クエリが 300K を超えるアイテムではない限り、データを取得するオプション **も表示されます**。 これにより、検索を編集しないか、データの収集を続行できます。

#### <a name="retrieve-data"></a>データの取得
データ取得の段階は、データ主体の個人データを含むすべてのファイル、電子メール、チャット、画像、その他のコンテンツ アイテムが取得され、レビューのために Azure BLOB ストレージ コンテナーにまとめる場合です。 データの量によっては、データの取得に数分または大幅に時間がかかる場合があります。 このステージが完了すると、要求は自動的にレビュー データの次のステージ **に移動します**。

#### <a name="review-data"></a>データの確認
 この段階で、投稿者は [データ収集] タブの下の結果を確認し、やり直し、タグの適用、メモの追加など、適用可能なすべてのタスクを実行する必要があります。 レビューが完了したら、[レビューの完了] **を選択します**。

#### <a name="generate-reports"></a>レポートの生成
レポートは、この段階で生成されます。 完了すると、[レポート] タブ **に表示されます** 。完成したファイルをエクスポートして、要求を行ったデータ主体に最終的な確認と配信を行うことができます。

#### <a name="close-the-request"></a>要求を閉じる
閉じた要求は、このサブジェクト権限要求を満たすためにすべての作業が完了したと示します。 収集されたデータとレポートはすべて、データ保持設定に従 [って保持されます](priva-settings.md#data-retention-periods)。

## <a name="step-2-optional-view-and-edit-search-queries"></a>手順 2 (オプション): 検索クエリの表示と編集

サブジェクト権限要求の背後にあるデータ検索に関する詳細情報を表示するには、[検索クエリの詳細の表示 **] を選択します**。 これにより、クエリを要約し、検出された内容に関する詳細を表示するウィンドウが開きます。

このクエリで返 **される** コンテンツの種類を確認するには、検索結果をプレビューするオプションがあります。 この検索のプロパティを変更する場合に、データの取得フェーズを開始していない場合は、[検索クエリの編集] オプション **を使用** できます。

検索クエリのガイド付き編集プロセスでは、データ主体の識別、検索フィルターと条件、およびデータを検索する場所 (Exchange、SharePoint、OneDrive、および/または Teams を含む) のプロパティを変更または追加できます。 これらのオプションを使用して、目的の特定のレベルに達します。 [保存] を押す前に、新しいクエリの最終バージョンを確認 **できます**。

検索クエリの編集が完了すると、以前の検索結果を置き換える新しい検索が実行されます。 これにより、[進行状況] セクションの状態 **が** 最初の手順である [データ推定] **にリセットされます**。 新しい検索の完了には最大 60 分かかる場合があります。 完了すると、要求の詳細ページに更新された結果が表示されます。

## <a name="step-3-review-data"></a>手順 3: データを確認する

この段階で、投稿者は [データ収集] タブで結果 **を確認する必要** があります。すべてのTeamsによるコンテンツ レビューを容易にするために、チャネルが自動的に設定されます。 詳細については [、「データ レビューで共同作業する](#collaborate-on-data-review) 」を参照してください。 データレビューステップの重要なタスクの概要を以下に示します。

#### <a name="mark-items-as-include-or-exclude-and-add-notes"></a>アイテムを [含める] または [除外] としてマークし、メモを追加する

検索によって返される識別されたアイテムの一覧を確認します。 アイテムを最終レポートの一部としてデータ主体に戻す必要がある場合は、項目の一覧の上部にあるコマンド バーの [含める] を選択します。 また、アイテムの一覧 **の右側** にあるコンテンツ レビュー領域で青い [含める] ボタンを選択することもできます。 [含める] **を選択** すると、メモを追加するオプションが表示されたフライアウト ウィンドウが表示されます。 完了したら、[送信] を **選択** してアイテムのレビュー状態を [含める] として保存 **します**。

アイテムが要求の一部として属していない場合は、コマンド バーの [除外] を選択するか、コンテンツ レビュー領域の [除外] ボタンを選択します。 アイテムを除外すると、データ主体に対して生成される最終レポートには含 [まれません](subject-rights-requests-reports.md)。

> [!NOTE]
> アイテムを除外としてマーク **する** 場合は、件名の権利要求に関連しない理由について、メモを正当な理由として追加する必要があります。 メモは内部的な目的のためであり、最終レポートには含まれません。

コンテンツが誤検知である場合は、[一致しない]  を選択し、フライアウト ウィンドウで [確認] を選択 **します**。 このアクションでは、最終レポートからファイルが除外され、アイテムに検索で検出されてはならないフラグが設定されます。

#### <a name="apply-tags"></a>Apply tags

タグを使用すると、さらに注意が必要なアイテムを特定できます。 Priva には、説明を設定できる 3 つの既定のタグ (フォローアップ、**削除**、および **更新**) が含まれます。 Priva には、名前と説明が可能な 2 つのカスタム タグも用意しています。

たとえば、データ レビュー中にコンテンツ アイテムを組織で保持する必要はないと判断した場合は、Delete タグを適用し、タグ付けされたファイルの一覧をエクスポートして、要求が完了したら、識別されたアイテムを元に戻して削除できます。

ユーザーが管理する 5 つの **タグ** 設定すべてのサブジェクト権限要求に適用されます。

**タグを追加または削除するには、**

- 要求の [データ収集] タブの一 **覧** からアイテムを選択します。
- リストの右側にあるアイテムプレビュー領域で、下の行の **[タグの適用** ] ボタンを選択します。 アイテム名の右側にある 3 つのドットを選択し、[タグの適用] **オプションを選択** することもできます。
- タグの一覧が表示されたフライアウト ウィンドウが表示されます。 アイテムに適用するタグの横にあるボックスをオンにします。 チェック ボックスをオフにすると、タグが削除されます。
- 完了したら、[保存] を **選択し**、タグの選択を保存し、フライアウト ウィンドウを閉じます。

**カスタム タグを追加したり、タグの説明を更新したりするには、次の方法を実行します。**
- [Subject Rights Requests] ページで、画面 **設定** 右上隅にある [権限の設定] を選択して、Priva の設定に移動します。
- [データ レビュー **タグ] ページに移動** し、説明を入力するタグと、カスタム タグの名前を選択します。 タグ設定について [詳しくは、次のページを参照してください](priva-settings.md#data-review-tags)。

**タグ付きアイテムのリストをエクスポートするには、次の方法を使用します。**
- サブジェクト権限要求 **の [データ収集** ] ページに移動します。
- アイテムの一覧の上で、下矢印アイコンを選択します。このアイコンの上にマウス ポインターを置くと、[ **エクスポート** ] と表示されます。
- 要求Excelによって収集されたアイテムのプロパティを示すファイルがダウンロードされます。 [タグ] **列を** 見つけて、タグでアイテムを識別して並べ替える。

#### <a name="use-the-annotate-command-to-redact-text"></a>[注釈] コマンドを使用してテキストを編集する
コンテンツ **レビュー領域の Annotate** コマンドを使用すると、インライン マークアップを作成し、コンテンツ アイテム内でデータを編集できます。 たとえば、別のデータ主体の個人情報も含む個人のファイルを含める必要がある場合は、コマンド バーの [描画] ボタンの下にある Area **redaction** を使用して、要求を行ったユーザーに関係しないすべての情報をブラックアウトできます。 編集が完了したら、[含める **] を選択** して、編集したファイルを要求に追加します。 注釈を使用すると、ファイルのコピーが作成され、Azure BLOB に格納されます。 元のファイルは変更されていないままで、元の場所に保存されます。

#### <a name="enter-notes-about-a-file"></a>ファイルに関するメモを入力する
アイテムのメモを追加または確認するには、その行からアイテムを選択し、右側のコンテンツ レビュー領域の [ファイル メモ] タブに移動します。 [ファイル メモの追加] **オプションを使用して** 、新しいコメントを作成することもできます。 ケース 全体レベルでメモを確認または追加するには、上記のメインの  [メモ] タブに移動し、[ケース メモの追加 **] を使用します**。 これらのメモは、要求で作業しているユーザーには表示されますが、最終レポートに含まれるか、データ主体と共有されません。

#### <a name="complete-the-review"></a>レビューを完了する

すべてのアイテムがレビューされ、その状態が [含める]、[除外]、または [一致しない] に設定されている場合は、要求の右上隅にある [レビューの完了] ボタンを選択して、レビュー 手順を終了します。 フライアウト ウィンドウにデータの概要が表示され、関連するメモが追加されます。 これらのメモは、内部レコード保持用であり、データ主体と共有されない。

[ **フライアウト] ウィンドウ** で [レビューの完了] を選択して、レビュー 手順を完了します。 決定の概要は、後で [レポート] タブで **提供** されます。

### <a name="collaborate-on-data-review"></a>データ レビューの共同作業

Priva は、グループがMicrosoft Teamsの権限要求に関して共同作業を行う場合に、グループ間の共同作業をサポートします。 新しい要求を作成すると、既定Teamsチャネルが自動的に作成され、要求に関連付けされます。 ここでは、要求について説明し、入力と投稿を安全に共有できます。 会話に参加するには、要求を開き、[共同作業者と **チャット] オプションを使用** します。 これにより、ユーザーがMicrosoft Teams権限要求のチーム サイトの全般チャネル内に配置されます。

チーム サイトを表示して投稿できるアクティブな共同作業者の一覧を確認するには、件名の要求内で [共同作業者] タブ **を開** きます。この要求で共同作業を行うユーザーを追加するには、[共同作業者を追加する **] オプションを選択します**。

サブジェクト権限要求を作成するときに Teams サイトを生成する既定の動作を変更するには、トップ ナビゲーションの **設定** に移動し、[Teams コラボレーション] を選択して **設定** を変更します。

また、件名の右の要求の右上にある [共有] オプションを使用して、Teams または電子メール経由でユーザーをループしたり、Priva のページへのリンクをコピーしたりすることもできます。 Teamsを使用して共有すると、アカウントで利用できる既存の Teams サイトとチャネルを選択できます。このケースへのリンクを、指定したメッセージと共に投稿します。

## <a name="step-4-close-the-request"></a>手順 4: 要求を閉じる

件名の要求を解決するために必要なすべてのアクションを実行した場合は、[要求を閉じる **] を選択します**。 これにより、最終的なレポートが作成され、[レポート] タブに **表示されます**。要求内のファイルの数によっては、完了に時間がかかる場合があります。
